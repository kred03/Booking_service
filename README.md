# Booking Service

Backend pet-project на ASP.NET Core Web API для управления пользователями, комнатами и бронированиями.

Проект сделан как учебный и портфолио-проект с упором на практику работы с REST API, Entity Framework Core, PostgreSQL, Swagger, валидацией входных данных и слоистой архитектурой.

## Что умеет проект

Сервис позволяет:

- создавать, получать, обновлять и удалять пользователей
- создавать, получать, обновлять и удалять комнаты
- создавать, получать, обновлять и удалять бронирования
- проверять пересечение бронирований по времени
- валидировать входные данные
- тестировать API через Swagger

## Бизнес-правила

Для бронирований реализованы базовые ограничения:

- нельзя создать бронь для несуществующего пользователя
- нельзя создать бронь для несуществующей комнаты
- дата окончания должна быть позже даты начала
- нельзя забронировать комнату на время, которое пересекается с уже существующей бронью этой же комнаты

## Стек

- C#
- ASP.NET Core Web API
- Entity Framework Core
- PostgreSQL
- Swagger / OpenAPI
- FluentValidation

## Архитектура

Решение разделено на 4 проекта:

### `Booking_Service.API`
Точка входа в приложение.

Содержит:
- контроллеры
- `Program.cs`
- конфигурацию Swagger
- подключение слоёв Application и Infrastructure

### `Booking_Service.Application`
Слой прикладной логики.

Содержит:
- DTO
- интерфейсы сервисов
- валидаторы
- вспомогательные application-модели

### `Booking_Service.Domain`
Доменный слой.

Содержит:
- сущности
- базовые доменные модели

### `Booking_Service.Infrastructure`
Инфраструктурный слой.

Содержит:
- `DbContext`
- конфигурации EF Core
- реализации сервисов
- регистрацию зависимостей
- работу с PostgreSQL
